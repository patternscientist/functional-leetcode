# 15. 3Sum

## Pattern

Sort, fix one index, then run a two-pointer two-sum scan on the suffix.

## Invariant

For a fixed i, l and r bound the remaining candidate range; sorted order tells which pointer can move after comparing the sum to zero.

## Code

for each non-duplicate i:
  l = i + 1; r = n - 1
  move l/r by sum, and skip duplicate l/r values after a hit

## Complexity

Time: O(n^2). Space: O(n) for sorting/array representation.

## Pitfall(s)

Canonicalize outputs in tests; the algorithm may produce valid triples in a different order. Skip duplicates at both the fixed index and after hits.
